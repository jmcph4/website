For some coursework I'm currently undertaking, I've had to read a paper entitled Learning to Fuzz from Symbolic Execution with Application to Smart Contracts[1] in order to prepare a seminar presentation for assessment.

As such, I've decided to typeset the handwritten notes I took as I read through the entirety of the paper.

2 - OVERVIEW

2.2 - Challenges of Fuzzing Smart Contracts

  • Smart contracts are stateful
  • Each transaction consists of
    1. Contract function (with arguments)
    2. Address of caller
    3. Gas sent to contract
  • Statefulness hinders automated testing
  • Often deep states
    • Require long sequences of transactions to reach them
      • Due to preconditions, etc.
    • Limits code coverage

Limited Coverage of Existing Fuzzers

  • Existing fuzzers fail to reach deep state
    • ECHIDNA
      • 57% coverage
      • Fails to cover functions with preconditions

Limited Coverage of Existing Symbolic Execution Tools

  • Reasoning about sequences of transactions symbolically
    • Block state explosion
      • Number of block states grows exponentially in number of analysed symbolic transactions, \(k\):
        • \[n_b\left(k\right)\in\mathcal{O}\left(n_b^k\right)\]
